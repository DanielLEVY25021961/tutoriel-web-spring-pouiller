types_application.html sous ${context}/static/

OBJECTIF : Montrer les différents types d'application et le rôle des divers fichiers XML.

Cette page HTML ${context}/static/types_application.html située sous le répertoire static sous le contexte (webapp) ne sert qu'a vérifier le fait que les ressources hors WEB-INF sont accessibles pour les navigateurs (css, images, ...).

- Nous formons l'hypothèse que tous les projets seront désormais développés avec MAVEN 3.5.0 (le 01/11/2017).
- Nous formons également l'hypothèse que tous les projets seront désormais développés avec JPA 2.1 (le 01/11/2017) si il y a persistance des données.
Même si l'ORM (Object-Relational-Mapping) utilisé au Ministère est Hibernate 5.2.11.Final (le 01/11/2017), nous utiliserons Java Persistence API (JPA) plutôt qu'une connexion directe via Hibernate.

Les principaux types d'application qui nous intéressent sont :

  1. Les applications STANDALONE (aussi appelées Desktop ou lourdes) SANS SPRING.
    Il s'agit d'applications développées en pur Java (JSE 1.8.144 le 01/11/2017) SANS conteneur SPRING.
    Les vues sont soit une console, soit développées avec AWT, SWING ou encore JavaFx.
  2. Les applications STANDALONE (aussi appelées Desktop ou lourdes) AVEC SPRING.
    Il s'agit d'applications développées en pur Java (JSE 1.8.144 le 01/11/2017) mais AVEC un conteneur SPRING MVC (5.0.1 le 01/11/2017).
    Les vues sont soit une console, soit développées avec AWT, SWING ou encore JavaFx.
    La structure applicative suit (du fait de SPRING) le modèle MVC2 : Vue - Modèle - Controller.
    La couche modèle comprend des SERVICES - METIER - DAO.
  3. Les applications WEB (aussi appelées Client léger ou légères) SANS SPRING.
    Il s'agit d'applications développées en pur J2EE (J2EE 1.8.144 le 01/11/2017) SANS conteneur SPRING.
    Les Controllers sont des SERVLETS déclarées dans un descripteur de déploiement web.xml.
    Les vues sont pour nos applications des JSP (Java Server Pages).
    Nous avons tendance à abandonner cette technologie au profit des applications WEB AVEC SPRING.
  4. Les applications WEB (aussi appelées Client léger ou légères) AVEC SPRING WEB MVC (5.0.1 le 01/11/2017).
    Il s'agit d'applications développées en J2EE (J2EE 1.8.144 le 01/11/2017) AVEC conteneur SPRING.
    Le Controller unique du MVC2 implémenté par SPRING est une SERVLET DispatcherServlet paramétré dans un fichier applicationContext.xml.
    Le descripteur de déploiement web.xml existe toujours et sert à déclarer le Controller MVC2 unique SPRING DispatcherServlet et à préciser quelles URL il doit intercepter.
    Les vues sont pour nos applications des JSP (Java Server Pages).

Rôle des fichiers XML dans une application WEB sous SPRING

SPRING apporte donc des fichiers xml en plus (par rapport à une application J2EE pur qui comporte un simple web.xml).
Néanmoins, SPRING est réellement rentable.

  • Le descripteur de déploiement d'application web web.xml est considérablement allégé par rapport au web.xml d'une application en J2EE pur dans laquelle on y déclarait tous les Controllers SERVLET.
    Il ne comporte plus que la déclaration du Controller unique MVC2 SPRING DispatcherServlet qui est paramétré par le contenu du fichier applicationContext.xml.
  • Le web.xml précise quelles URL sont interceptées par le Controller MVC2 SPRING.
  • Le web.xml se charge également de la redirection (dans les welcome-files) vers une ressource statique du type index.html directement sous le webapp.
    Cette index.html possède une redirection HTML vers la page d'accueil de l'application (en passant par le Controller Spring).
    On peut ainsi accéder à la page d'accueil de l'application dès que l'on a tapé l'URL du contexte du site dans un navigateur.
  • Le descripteur de SPRING applicationContext.xml définit tous les beans et comportements de SPRING.
    En particulier :
    • InternalResourceViewResolver se charge de l'aiguillage vers les vues (jsp).
    • <context:component-scan base-package="levy.daniel.application"/> permet l'auto-detection par SPRING de toutes les classes annotées dans le package indiqué.
    • <mvc:annotation-driven> permet la configuration par défaut de SPRING WEB MVC.
    • LocalContainerEntityManagerFactoryBean se charge de l'instanciation de la Factory d'EntityManager chargés de gérer la persistance des données.
    • TransactionManager se charge de la gestion des transactions de persistance.
    • Le bean d'id= dataSource permet de définir ou de référencer la source des données (définition directe si on ne passe pas par un serveur web, référencement en utilisant un annuaire JNDI si la dataSource est gérée par un serveur WEB comme TOMCAT).

Implémentation des ressources de cette page ${context}/static/indexStatic.html avec chemin ABSOLU PAR RAPPORT au CONTEXT

Cette page utilise une css : ${context}/static/css/style_index.css.

La css est référencée en ABSOLU href="css/style_index.css". En absolu signifie par rapport au contexte.

L'absence de "/" au début de l'URL permet au serveur de résoudre le contexte : http://host:8080/context/
href="css/style_index.css" permet de décrire le chemin de la ressource css PAR RAPPORT A LA POSITION DE LA PAGE ACTUELLE à concaténer au contexte lors de la résolution par le serveur.
